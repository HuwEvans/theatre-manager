# Changelog

## Version 3.7.17
Color contrast accessibility improvements.
- Fixed glass button: Changed white text on transparent white to dark text on subtle dark overlay
- Fixed empty rating stars: Changed white color to dark gray at 30% opacity
- Improves WCAG AAA accessibility compliance
- All text now has proper contrast with backgrounds

## Version 3.7.16
Cast image display improvement.
- Fixed cast member images that were overflowing and cropping
- Changed object-fit from cover to contain to show entire picture
- Added light gray background (#f0f0f0) for clean appearance
- Images remain fully responsive and scale with castcols parameter

## Version 3.7.15
Critical bug fix for cast grid layout.
- Fixed critical issue where castwithbio field displayed only one column with very large images
- CSS rule was using display: flex which overrode the grid layout
- Restored proper display: grid with grid-template-columns using --cast-cols CSS variable
- Cast grid now properly displays multiple columns respecting the castcols parameter

## Version 3.7.14
Comprehensive fixes: custom post type capabilities, responsive image scaling, castcols honoring, and field alignment inheritance.
- Fixed custom post type capability issue preventing users from editing posts
  - Added 'capability_type' => 'post' and 'map_meta_cap' => true to all 10 CPTs
  - Resolves "Sorry, you are not allowed to edit posts in this post type" error
  - Applies to: show, cast, season, venue, award, advertiser, board_member, sponsor, testimonial, contributor
- Enhanced cast member image sizing for responsive display
  - Images now scale based on castcols parameter: 300px (1 col) to 210px (6 cols)
  - Uses formula: height = calc(300px - (castcols * 15px)) with max() fallback
  - Added aspect-ratio: 3/4 for consistent portrait headshots
  - Desktop: 300-210px, Tablet: 270-200px, Mobile: 220-160px
  - All images remain fully responsive to container width
- Fixed castcols parameter to be honored across all responsive breakpoints (previously ignored on mobile)
  - Desktop (>1024px): Displays castcols columns as specified
  - Tablet (1024px-768px): Displays min(castcols, 2) columns for readability
  - Mobile (<768px): Displays min(castcols, 1) column (single column minimum for usability)
- Enhanced all shortcode fields to inherit alignment and text styling from surrounding page context.
- All fields now respect Gutenberg block alignment settings (center/left/right/justify).
- Extended support for Beaver Builder alignment settings and theme-specific alignment classes.
- Fields inherit all text properties: font-family, font-size, color, font-weight, line-height, letter-spacing, text-transform, direction.
- Nested elements (cast lists, venue info) fully inherit parent styling and alignment.
- Images automatically center within centered content using display: block and auto margins.
- Enhanced CSS sibling selectors to detect:
  - Gutenberg: has-text-align-* and wp-block-paragraph classes
  - Beaver Builder: fl-col text alignment classes
  - Theme-specific: centered, center, and other common alignment classes
- Support for blocks separated by horizontal rules (<hr>).
- Cast with bio grid now centers when parent is centered.
- All text settings cascade properly from page/block context to all shortcode fields.
- Updated PHP comments to document castcols behavior at different breakpoints.
- Version updated to 3.7.14.
